# Database CLI Access

Users frequently forget how to get a shell into the databases. When they ask, give them these commands directly.

## picsure-db (MySQL)

```bash
MYSQL_PWD="$DB_ROOT_PASSWORD" docker compose exec -e MYSQL_PWD picsure-db mysql -u root picsure
```

- Service name: `picsure-db`
- Databases: `picsure` (Wildfly/API), `auth` (psama)
- Root password lives in `.env` as `DB_ROOT_PASSWORD` — do not print it. The HOST shell expands the env-prefix assignment into docker's environment (never into its argv), and the bare `-e MYSQL_PWD` forwards it by name into the container — so the value never appears in `ps`. Do NOT write `-e MYSQL_PWD="$DB_ROOT_PASSWORD"`: that expands the password into the host-visible docker argv.

For `DB_MODE=remote`, use your database client against `DB_HOST`/`DB_PORT`
instead of `docker compose exec picsure-db`.

To connect without relying on host env expansion, exec into the container first:

```bash
docker compose exec picsure-db bash
mysql -u root -p picsure     # will prompt for password
```

## dictionary-db (PostgreSQL)

```bash
docker compose exec dictionary-db psql -U postgres dictionary
```

- Service name: `dictionary-db`
- Default database: `dictionary`

## Notes
- Credentials are generated by `init.sh` and stored in `.env`. If passwords drift, see the "Database auth errors" section of `README.md` for the reset procedure (it destroys data).
- Do not read `.env` to grab the password — instruct the user or rely on the container's own environment.
